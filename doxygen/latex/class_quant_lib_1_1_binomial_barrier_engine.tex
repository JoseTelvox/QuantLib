\subsubsection*{template$<$class T, class D$>$\newline
class Quant\+Lib\+::\+Binomial\+Barrier\+Engine$<$ T, D $>$}
Pricing engine for barrier options using binomial trees. 

\begin{DoxyNote}{Note}
Timesteps for Cox-\/\+Ross-\/\+Rubinstein trees are adjusted using Boyle and Lau algorithm. See Journal of Derivatives, 1/1994, "{}\+Bumping up against the barrier with the binomial method"{}
\end{DoxyNote}
\begin{DoxyRefDesc}{Test}
\item[\mbox{\hyperlink{test__test000106}{Test}}]the correctness of the returned values is tested by checking it against analytic european results. \end{DoxyRefDesc}

The max\+Time\+Steps parameter is used to limit time\+Steps when using Boyle-\/\+Lau optimization. If zero (the default) the maximum number of steps is calculated by an heuristic\+: anything when \texorpdfstring{$<$}{<} 1000, otherwise no more than 5\texorpdfstring{$\ast$}{*}time\+Steps. If max\+Time\+Steps is equal to time\+Steps, Boyle-\/\+Lau is disabled. Likewise if the lattice is not \doxylink{class_quant_lib_1_1_cox_ross_rubinstein}{Cox\+Ross\+Rubinstein} Boyle-\/\+Lau is disabled and max\+Time\+Steps ignored.
