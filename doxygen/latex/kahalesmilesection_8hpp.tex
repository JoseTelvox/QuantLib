Arbitrage free smile section using a C\texorpdfstring{$^\wedge$}{\string^}1 inter-\/ and extrapolation method proposed by Kahale, see \href{http://www.risk.net/data/Pay_per_view/risk/technical/2004/0504_tech_option2.pdf}{\texttt{ http\+://www.\+risk.\+net/data/\+Pay\+\_\+per\+\_\+view/risk/technical/2004/0504\+\_\+tech\+\_\+option2.\+pdf}} Exponential extrapolation for high strikes can be used alternatively to avoid a too slowly decreasing call price function. Note that in the leftmost interval and right from the last grid point the input smile is always replaced by the extrapolating functional forms, so if you are sure that the input smile is globally arbitrage free and you do not want to change it in these strike regions you should not use this class at all. Input smile sections with a shift are handled accordingly, normal input smile section are not possible though.
